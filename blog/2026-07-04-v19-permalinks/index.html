2026-07-04 — v19 shareable permalinks

July 4, 2026

This slice ships the viral wedge: “Share this view” now copies a hash permalink that recreates the exact sample view (zoom window, strand, search, selection, edits, trim mode, and track visibility) fully client-side. No uploads, no backend, no hidden sync.

Permalink hydration now boots directly from URL hash state for sample traces, while local-file permalinks degrade honestly with a clear “re-open local file” message because trace bytes remain private. URL-size guardrails are explicit: sample + view state always shares, but oversized state is blocked with user feedback instead of emitting a broken link.

Validation added exact-value tests for encode/decode round-trip, oversize handling, known permalink state restoration (base/zoom/revcomp), and clipboard copy behavior.
