2026-07-07 — sequence-file interop hardening plan

July 7, 2026

This docs-only slice turns the SnapGene migration blocker into an implementation-ready sequence of small PRs. It records the current parser/export gaps, fixture requirements, and acceptance evidence needed before claiming robust SnapGene .dna, GenBank, FASTA, AB1, and SCF interop.

What changed

  • Added a durable research artifact for sequence-file interop hardening, including current parser state, competitor migration expectations, record-model requirements, fixture provenance needs, and parser-risk notes.
  • Added a step-by-step implementation plan that splits the work into CI-green PRs: binary parser errors, sequence-record/FASTA foundation, GenBank parse/write, SnapGene .dna fixture/parser, ABI corpus expansion, and worker-backed UX integration.
  • Kept this PR scoped to planning artifacts so follow-on implementation agents can work in parallel without changing runtime behavior.

Validation plan for the follow-on wave

The planned acceptance gate is intentionally test-first: a real public Addgene .dna fixture must round-trip through GenBank with zero feature, primer, or note loss; at least 100 real ABI files across at least three instrument models must parse with expected base-call and quality sentinels; and corrupt/truncated trace files must return graceful parser errors through the worker instead of crashing the app.
