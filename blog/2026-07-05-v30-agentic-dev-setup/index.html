This pass lays down the written runway for focused future agent work: a research note, an implementation plan, a first ADR, and a LEARNINGS file that captures the repo conventions future contributors should not have to rediscover.

Why it matters

sanger-viewer has enough surface area now that “just work on the app” is too blunt. Capturing those habits in durable repo docs should make future PRs smaller, easier to review, and safer for the live GitHub Pages site.
