2026-07-05 — v30 agentic development setup: domain guidance, Playwright-ready setup, and durable learnings

July 5, 2026

This pass makes the repository friendlier for the next small, careful change. The guidance now says plainly what sanger-viewer is: a private, client-side Sanger chromatogram workbench for bench scientists, deployed on GitHub Pages and held to true-user browser validation.

What changed

  • AGENTS.md and .github/copilot-instructions.md now document the TypeScript/Vite/GitHub Pages stack, exact validation commands, privacy model, UX screenshot gate, small-PR rhythm, devlog expectations, ADR expectations, and Definition of Done.
  • .github/workflows/copilot-setup-steps.yml installs npm dependencies and Chromium with npx playwright install --with-deps chromium inside the required copilot-setup-steps job, capped at 59 minutes.
  • docs/adr/0001-agentic-development-setup.md records why this setup belongs in the repo instead of being rediscovered in each session.
  • LEARNINGS.md starts a durable notebook for bench-scientist UX, client-side privacy, Playwright setup, UX screenshots, and documentation conventions.

UX assessment

Is this great UX? This is not a product UI change, so no chromatogram screen moved. It should improve future UX work indirectly by making the desktop, tablet, and mobile screenshot gate harder to miss and by asking every change to be judged from a bench scientist's point of view.

What to improve next: The next setup pass should add focused role-specific reviewer and builder definitions once that PR owns the custom-agent files. Those definitions should point back to the same Definition of Done so specialists do not drift from the repository standard.
