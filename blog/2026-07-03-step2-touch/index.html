tablet and touch pass: pinch, pan, tap, and a saner layout

July 3, 2026

This was the first "make it feel like a real app on glass" pass, and it changed the personality of the viewer more than I expected. The big win is that the chromatogram now listens to pointer events instead of a mouse-only path, which means one-finger drag pans the trace, two-finger pinch-to-zoom works, and a clean tap can inspect a base instead of leaving touch users out in the cold.

I kept the implementation intentionally boring: reuse the existing canvas zoom/pan primitives, keep the viewport readout as the source of truth in tests, and let Playwright exercise the gestures on an emulated iPad project. That gave us a nice side benefit too: the existing fixture-driven E2E flow now runs on both desktop and tablet form factors instead of pretending desktop coverage was enough.

On the layout side, this step is mostly about not fighting the screen. Buttons are now touch-sized, the control row has more breathing room when it wraps, and the shell padding scales down on narrower widths so the canvas keeps the attention instead of the margins. The viewport meta tag was already correct, so I left that alone rather than churn a file for no reason.

Next up is the friction pass: clearer empty states, sample-file loading, better drag-and-drop affordance, visible loading/error handling, and accessibility polish so the app feels calm and obvious before we push harder on performance and pro-grade feature parity.
