2026-07-06 — role-specialized UX-leadership enablement

July 6, 2026

This docs-and-guidance pass gives the next UX-leadership wave a shared operating contract. It connects the competitor benchmark, the current UX-gallery assessment, and the no-regression test gates to six focused roles: UX design, accessibility, performance, test authoring, code review, and devlog writing.

What shipped

  • A refreshed enablement research note under docs/research/ comparing SnapGene, Benchling, Chromas, FinchTV, and ApE against the current sanger-viewer gallery evidence.
  • A focused implementation plan under docs/plans/ with role briefs that can be mirrored into role files by a maintainer with access to that directory.
  • Repository guidance updates in AGENTS.md and .github/copilot-instructions.md so future work keeps the UX-gallery gate, WCAG 2.1 AA, performance budgets, design-token reuse, and feature preservation front and centre.
  • A deterministic docs test that fails if the enablement artifacts drop the required roles, competitor benchmark, screenshot gate, validation contract, or attribution-sensitive wording guard.

Screenshot UX assessment

No runtime UI changed in this PR, so the current screenshots remain the v28/v29 gallery baseline. The written assessment rates the existing first-run/dropzone, hero chromatogram hierarchy, design-system cohesion, accessibility, performance, and power-preservation posture, then turns each gap into a testable backlog item for future visual PRs.
